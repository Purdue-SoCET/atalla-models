## 1. Architecture Overview

```
PyTorch nn.Module
    Ōöé
    Ōö£ŌöĆ atalla-graph/        FX graph capture, op normalization, tiling, code generation
    Ōö£ŌöĆ aihw-ppci-compiler/  AtallaC ŌåÆ assembly compiler
    ŌööŌöĆ functional_sim/      Assembler, .in builder, emulator
```

**Two execution paths:**

```
PATH 1: HAND-WRITTEN ASSEMBLY (ground truth, standalone validation)
  build_relu.py ŌöĆŌöĆŌ¢║ encode + DRAMWriter ŌöĆŌöĆŌ¢║ complete .in ŌöĆŌöĆŌ¢║ functional_sim

PATH 2: C COMPILER (primary graph pipeline)
  c_emitter.py ŌöĆŌöĆŌ¢║ .c kernel ŌöĆŌöĆŌ¢║ ppci ŌöĆŌöĆŌ¢║ .s ŌöĆŌöĆŌ¢║ build_compiler ŌöĆŌöĆŌ¢║ instruction section
  DRAMWriter ŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌ¢║ data section
                                                                         Ōöé
                                                            render_testfile() ŌåÆ .in ŌåÆ functional_sim
```

`build_compiler.py` produces the instruction section. The data section (config at `0x3C` + tensors) is generated by `c_emitter.py`'s `DRAMWriter`. `render_testfile()` combines both into a `.in` file.

**Passthrough ops** (`flatten`, `transpose`, `dropout`) need no kernel ŌĆö they're pure tensor reshapes with no computation. The Python orchestrator just reshapes the numpy array in `activation_cache` and passes it to the next layer. No `.in` file is generated.

**Current status:** Only `mul` (BasicModule residual) still falls back to NumPy. AlexNet has **zero** NumPy fallbacks.

## 2. atalla-graph

### 2.1 Graph Capture (`graph/fx_capture.py`)

1. Converts weights to `bfloat16`
2. `torch.fx.symbolic_trace(model)` ŌåÆ flat FX graph
3. `ShapeProp` propagates shapes
4. Tags each node with `node.meta["atalla_op"]` via lookup tables:

| Source | Example | Maps to |
|--------|---------|---------|
| `_OP_MAP` (call_function) | `torch.matmul`, `F.relu`, `F.conv2d` | `"matmul"`, `"relu"`, `"conv"` |
| `_METHOD_MAP` (call_method) | `.flatten()`, `.add()`, `.transpose()` | `"flatten"`, `"add"`, `"flatten"` |
| `_MODULE_MAP` (call_module) | `nn.Conv2d`, `nn.MaxPool2d` | `"conv"`, `"maxpool"` |

`torch.matmul` and `torch.ops.aten.matmul.default` both map to `"matmul"` ŌĆö the aten variant appears when `lower_linear_modules` decomposes `nn.Linear` into `transpose + matmul + add`.

### 2.2 Tile Planning (`graph/tile_planner.py`)

Hardware: VL=32, systolic tile=32├Ś32, scratchpad=32 slots/bank, bf16.

| Op | Planning |
|----|----------|
| **Conv** | im2col: M=Ho├ŚWo, K=R├ŚS├ŚC_in, N=C_out, tiles=ceil(dim/32) |
| **Linear/Matmul** | M, K, N from shapes, tiles=ceil(dim/32) |
| **ReLU/Add/Mul** | total_elements, width=min(total, 32) |
| **Softmax** | length from last dimension |
| **MaxPool** | H, W, C, pool_size, stride ŌåÆ H_out, W_out |
| **Flatten/Transpose** | Passthrough, no tiling |

DRAM addresses start at `0x1000`, 2KB-aligned. `0x00`ŌĆō`0x3C` reserved for ADDR_TABLE.

### 2.3 Code Generation (`codegen/c_emitter.py`)

`emit_node()` dispatch:

| Op | Handler | Backend |
|----|---------|---------|
| conv, linear, matmul | `emit_conv/linear/matmul` | On-chip (GEMM kernel) |
| relu | `emit_relu` | On-chip |
| softmax | `emit_softmax` | On-chip |
| maxpool | `emit_maxpool` | On-chip (vertical max) |
| add | `emit_add` | On-chip (dual scratchpad) |
| mul | `emit_mul` | NumPy fallback |
| adaptive_avg_pool | `emit_adaptive_avg_pool` | NumPy fallback |

Each `emit_*` returns a `LayerEmission` with: `c_source` (AtallaC), `dram` (DRAMWriter with config + tensors), `output_addr`, `output_elements`.

**Kernel generators** in `atalla-graph/kernels/`:

| Generator | Technique |
|-----------|-----------|
| `gemm.py` | Tiled GEMM: `scpad_load/store`, `vector_load/store`, `load_weights`, `gemm()` |
| `relu.py` | `make_mask("<", v, zero)` ŌåÆ `vec_op_masked("*", v, 0.0, mask)` |
| `softmax.py` | `RMAX` ŌåÆ shift ŌåÆ `EXP` ŌåÆ `RSUM` ŌåÆ `1.0 / sum` |
| `maxpool.py` | `make_mask(">")` ŌåÆ blend (vertical max) |
| `add.py` | Dual scratchpad: AŌåÆSP0, BŌåÆSP1, `vec_op_masked("+")` |

All use first-class intrinsics. Only `lw_s`, `li_s`, `halt` remain as inline asm.

### 2.4 `run_graph.py`

Modes: `validate` (emulator execution + PyTorch comparison) and `schedule` (C codegen via Vihaan's `generate_schedule`).

## 3. aihw-ppci-compiler

### 3.1 AtallaC Extensions

| Feature | Syntax |
|---------|--------|
| Vector type | `vec v;` (32-element register) |
| GEMM | `gemm(a_row, c_row, mask)` |
| Masked ops | `vec_op_masked("+", a, b, mask)` |
| Mask creation | `make_mask(">", a, b, mask)` |
| Intrinsics | `scpad_load/store`, `vector_load/store`, `load_weights`, `sqrt` |
| Implicit division | `1.0 / x` ŌåÆ `rcp_bf + mul_bf` |
| Type casts | `float(x)` ŌåÆ `stbf_s`, `(int)y` ŌåÆ `bfts_s` |

### 3.2 Compilation Flow

```
.c ŌåÆ ppci frontend ŌåÆ IR ŌåÆ instruction selection ŌåÆ register allocation ŌåÆ .s
.s ŌåÆ build_compiler: parse ŌåÆ bootstrap ŌåÆ expand li_s ŌåÆ VLIW schedule ŌåÆ encode ŌåÆ .in
```

### 3.3 Limitations

- **Stack frame overflow**: Many live vectors exceed 12-bit offset range. Workaround: reduce live vector count.
- **No vector SQRT / shift.vi**: Only scalar `sqrt()` works. Blocks vector-width ops.
- **Config loads require inline asm**: No intrinsic for `lw_s`, `li_s`, `halt`.

## 4. functional_sim

### 4.1 Architecture

| Component | Description |
|-----------|-------------|
| Scalar regs | 34 regs; x0=zero, x2=stack ptr, x33=spill base |
| Mask regs | 16 regs; m0 hardwired to 0xFFFFFFFF |
| Vector regs | 64 regs ├Ś 32 bf16 elements |
| Scratchpads | SP0, SP1: 32 slots ├Ś 32 bf16 values |
| Memory | DRAM, dictionary-based, 32-bit words at 2-byte stride |

### 4.2 Encoding Toolchain

**`build.py`** (Sahil's): `encode_instruction` ŌåÆ `assemble_file` ŌåÆ `emit_test_format` (1 instr/packet, not dependency-aware) + `DRAMWriter` + `render_testfile`. Used by `build_*.py` scripts.

**`build_compiler.py`**: `compile_asm()` ŌĆö parse ŌåÆ bootstrap ŌåÆ li_s expansion ŌåÆ **dependency-aware VLIW scheduling** ŌåÆ encode. Packs multiple instructions per packet. Used by the graph pipeline.

Both produce the same `.in` format. `build_compiler` achieves better slot utilization.

### 4.3 ADDR_TABLE Convention

Kernels read parameters from address **0x3C**:

| Layout | Fields |
|--------|--------|
| GEMM/Conv/Linear | A_GMEM, W_GMEM, C_GMEM, M, N, K, tile counts |
| ReLU/Softmax | IN_GMEM, OUT_GMEM |
| Add | A_GMEM, B_GMEM, C_GMEM |

### 4.4 Emulator Fixes We Made

| Fix | What |
|-----|------|
| m0 = all-ones | Hardware spec; upstream returns 0 |
| BF16 register convention | `bf16_reg_to_f32`/`f32_to_bf16_reg` for compiler's 16-bit-in-lower-half storage |
| `gemm.vv` order | `gemm_weights @ v_in + v_acc` (W^T @ v_in) |
| `lw.vi` weight reset | Clear `gemm_weights` between GEMM tiles |
| Stack/DRAM overlap | Dynamic stack base above all DRAM data |
| `stbf.s`/`bfts.s` | Proper BF16Ōåöfloat conversion |
| 3-reg SDMA decode | Bit 34 flag for compiler's packed control register format |
| `sqrt.bf` support | New opcode + scalar ALU implementation |

## 5. Kernel Library

Three levels, validated bottom-up:

| Level | Location | Purpose |
|-------|----------|---------|
| Ground truth | `functional_sim/build_*.py` | Hand-written asm + DRAMWriter data, run on emulator |
| Reference C | `aihw-ppci-compiler/atalla_tests/kernels/*.c` | AtallaC with intrinsics, compiled via ppci |
| Generators | `atalla-graph/kernels/*.py` | Parameterized AtallaC for arbitrary tensor shapes |

### GEMM Algorithm (used by conv, linear, matmul)

```
for each tile triple (mi, ni, ki):
  scpad.ld A_tile, W_tile, C_tile
  for w in 0..TILE: lw.vi w          # preload weights into systolic array
  for r in 0..TILE:
    vreg.ld a_row, c_row
    gemm.vv c_row, a_row, mask       # c += a * W
    vreg.st c_row
  scpad.st C_tile
```

## 6. Full Pipeline Integration

### 6.1 Execution Flow

```python
gm = build_graph(model, input)  # FX trace ŌåÆ lower_linear_modules ŌåÆ ShapeProp ŌåÆ normalize ŌåÆ tile plan

for node in gm.graph.nodes:
    if passthrough(node):                    # flatten, transpose, dropout
        activation_cache[node.name] = reshape(prev)
        continue

    emission = emit_node(node, gm, activation_cache)

    if emission.skip_emulator:               # NumPy fallback
        activation_cache[node.name] = emission.numpy_result
    else:                                    # on-chip
        compile_and_assemble(emission)       # .c ŌåÆ ppci ŌåÆ .s ŌåÆ build_compiler ŌåÆ instr section
        write(.in = instr_section + DRAMWriter_data)
        mem = run_emulator(.in)
        activation_cache[node.name] = read_bf16(mem, output_addr)

compare(activation_cache[final], pytorch_reference)
```

Each layer is a standalone emulator invocation with fresh state. Stack pointer (`x2`) placed dynamically above all DRAM data.

### 6.2 How `.data` Propagates

1. `emit_<op>()` creates a `DRAMWriter` ŌåÆ writes config at 0x3C + input/weight tensors as BF16
2. `compile_and_assemble()` produces the instruction section only (strips `.data` from `build_compiler` output)
3. `render_in_file()` combines instruction section + `DRAMWriter.render_data_mem()` ŌåÆ complete `.in`
4. Emulator loads `.in`, kernel reads config via `lw_s`, uses SDMA to move tiles between DRAM and scratchpad
5. `_read_bf16(mem, output_addr)` extracts output from emulator memory
6. Output goes into `activation_cache` ŌåÆ next layer's `emit_<op>()` serializes it as input into a fresh `DRAMWriter`

Validated and working for all kernels and full AlexNet (21 emulated nodes, 0 NumPy).

## 7. AlexNet Walkthrough

### AlexNetSmall (`model/alexnet_small.py`)

`scale` parameter controls channel widths: `sc(c) = max(1, int(c * scale))`. At `scale=0.01`: channels 64ŌåÆ1, 192ŌåÆ1, 384ŌåÆ3, 256ŌåÆ2. Input 32├Ś32. No AdaptiveAvgPool or Dropout.

Vihaan's full AlexNet (`model/alexnet.py`) uses `torchvision.models.alexnet()`: 224├Ś224 input, full channel counts, includes AdaptiveAvgPool2d and Dropout. All ops we support except AdaptiveAvgPool2d (which is identity for AlexNet's spatial dims) and Dropout (removed at inference).

### FX Graph (AlexNetSmall, scale=0.01)

After `lower_linear_modules`: **42 nodes** ŌĆö 21 emulated (5 conv, 7 relu, 3 matmul, 3 maxpool, 3 add), 0 NumPy, 4 passthrough (flatten + transpose).

### Conv1 Example

`Conv2d(3, 1, 3├Ś3, stride=1, pad=1)` on `(1, 3, 32, 32)`:
- im2col: M=1024, K=27, N=1 ŌåÆ `_gemm_c(1024, 1, 27)` ŌåÆ 32 tiles

## 8. Validation Results

The functional simulator is not cycle-accurate. Primary efficiency metric is **VLIW slot utilization** (`instructions / (packets ├Ś 4)`).

### BasicModule (dim=32, depth=2)

9 emulated (cos=1.0 each), 1 numpy (mul), 1 passthrough. **End-to-end cos=0.9999.**

### AlexNetSmall (scale=0.01)

21 emulated, 0 numpy, 4 passthrough. **End-to-end cos=0.969.** BF16 drift through 5 conv + 3 FC layers. No NaN.
